Own System for Variables

Dialog Maker has it's own system for setting, changing, unsetting and reading variables. The main thing which make me to do this system is that I wanted Dialog Maker to save it's variables in another file. Mirc saves it's variables in Remote.ini and Dialog Maker saves it's in Variables.ini which is in DialogMaker\Data\. And that's why if you unset all mIRC variables or delete the file Remote.ini Dialog Maker's variables and options are not deleted. Commands which are used for working with Dialog Maker variables are (These commands can be used only by Dialog Maker. I mention them here only because you may like them and after that I can put them into a snippet):
1. Set Variable

Command: /var.set [Section] [Variable] [Value]

Explanation: [Section] is the section of the INI file where the variable will be set. [Variable] is the variable which you want to set/change value of.

Hint: If [Value] is not setted, the command unsets the following variable.

Example: /var.set Global Welcome 5

The example will set the variable Welcome to value 5 in the section Global of the INI file.
2. Unset Variable

Commands:

  1. 1. /var.unset [Section] [Variable1], [Variable2], [Variable3], [Variable4], ...
  2. 2. /var.unset [Section] [Variable]*
  3. 3. /var.unset [Section] *[Variable]
  4. 4. /var.unset [Section]

1. /var.unset [Section] [Variable1], [Variable2], [Variable3], [Variable4]

Explanation: [Variable1-X] is the the variable which you want to delete from the variables list.

Hint: You can unset a lot of variables only with one command. Just after every entered variable type comma (,).

Example: /var.unset First Welcome, Hello, Color, Options

The example will unset variables Welcome, Hello, Color and Options from the section First of the INI file.

2. /var.unset [Section] [Variable]*

Explanation: This command deletes all variables starting with [Variable] from the variables list. The * at the end means that all variables starting with [Variable] will be deleted.

Hint: You can use this command only for one variable.

Example: /var.unset Registry Key*

The example will unset all variables in the section Registry of the INI file which start with the string Key.

3. /var.unset [Section] *[Variable]

Explanation: This command deletes all variables ending on [Variable] from the variables list. The * at the beginning means that all variables ending on [Variable] will be deleted.

Hint: You can use this command only for one variable.

Example: /var.unset Variables *time

The example will unset all variables in the section Variables of the INI file which end on the string time.

4. /var.unset [Section]

Explanation: This command unsets all variables in specified section of the INI file.

Example: /var.unset Common

The example will unset all variables in the section Common of the INI file.
3. Increase Variable

Command: /var.inc [Section] [Variable] [Value]

Explanation: [Variable] is the variable which you want to increase. The [Variable] will be increased with entered [Value].

Hint: If [Value] is not entered, the command will increase the following variable with 1.

Example: /var.inc Vars Times

The example will increase variable Time's value with 1. The variable is in the Vars section of the INI file.
4. Decrease Variable

Command: /var.dec [Section] [Variable] [Value]

Explanation: [Variable] is the variable which you want to decrease. The [Variable] will be decreased with entered [Value].

Hint: If [Value] is not entered, the command will decrease the following variable with 1.

Example: /var.dec Beginning Start 3

The example will decrease variable Start's value with 3. The variable is in the Beginning section of the INI file.
5. Read Variable

Command: /var.read [Section] [Variable] or $var.read([Section],[Variable])

Explanation: [Variable] is the variable which will be read from the variables list.

Hint: $var.read([Variable]) often is used when you have some command before it.

Example: /echo $var.read(Info,Nick)

The example will read and show the variable Nick from the variables list. The variable will be read from the section Info from the INI file.
