FAQ (Frequently Asked Questions)
If I close Visual Editor how to open it again?
If you close Visual Editor first open Options dialog (In main window Tools >>> Options), next in the 'Editors' menu select the item 'Visual Editor' and finally click 'Open' to open it again.There is an easier way too - Click on 'Save' button (Or 'Save' in the Control Editor). This, except opening Visual Editor, saves the project. If you want to open only Visual Editor without saving use the first (longer) way.
Can I make my own skins for Dialog Maker?

Of course you can. To make your own toolbar buttons skin just create new folder in the Dialog Maker directory\Data\Skins\ with the name of the skin. Put in it files with same names as files in the Default skin directory (Dialog Maker directory\Data\Skins\Default\):

Clear.ico, Code.ico, Control.ico,Cut.ico, Copy.ico,Delete.ico, DialogCode.ico, Goto.ico, Help.ico, Language.ico, New.ico, Open.ico, FromFile.ico, Options.ico, Paste.ico, Patterns.ico,Brackets, Preview.ico, Print.ico, Save.ico, Tools.ico, Total.ico, OK.ico, Error.ico and Update.ico. Every icon is for button with same or similiar name from the main and Code Editor's toolbar. Logo.jpg file is the file which is loaded as a background picture in Options window. Default picture's resolution is 372 x 235 pixels. Some of the icons are used for two or more thing at a same time. This is used to reduce the skin size. These are the icons which are used for more than one thing: Save.ico, Copy.ico and Paste.ico.

If you want to change the background of the main toolbar on your skin, just put a bitmap file with name Toolbar.bmp in your skin folder. This file will be loaded as a toolbar background (Tiled) instead of the default toolbar background color. I recommend the resolution of the picture to be 32x32. Of course it can be bigger or smaller but if it's bigger this will increase the skin size and if it's smaller this may increase the skin's loading time.
Is it possible for me to make my own modules?
Yes, it is. You have only to create a new file [filename].mod, put it in the Modules folder in Dialog Maker's one and to follow the pattern in the default module which comes with Dialog Maker - Invalid ID Checked. (The pattern shows that you must have two section like in a .ini file - [Information] and [Module]. To fill them follow the pattern, as I wrote before)
